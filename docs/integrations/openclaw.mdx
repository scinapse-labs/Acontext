---
title: "OpenClaw"
description: "Skill memory plugin for OpenClaw agents — auto-capture, auto-recall, auto-learn"
---

The `@acontext/openclaw` plugin gives OpenClaw agents cross-session skill memory. It captures conversations, extracts tasks, distills reusable Markdown skills, and recalls them in future sessions — all automatically.

## Quick Start

<Steps>
<Step title="Install the plugin">
```bash
openclaw plugins install @acontext/openclaw
```
</Step>

<Step title="Set your API key">

Get an API key from [dash.acontext.io](https://dash.acontext.io/) and export it:

```bash
export ACONTEXT_API_KEY=sk-ac-your-api-key
```
</Step>

<Step title="Configure openclaw.json">

Add the Acontext plugin to your `openclaw.json` config:

```json5
// plugins.entries
"acontext": {
  "enabled": true,
  "config": {
    "apiKey": "${ACONTEXT_API_KEY}",
    "userId": "your-user-id"
  }
}
```
</Step>

<Step title="Start the gateway">
```bash
openclaw gateway
```

Your agent now has skill memory. Every conversation is captured, tasks are extracted, and skills are distilled for future recall.
</Step>
</Steps>

## How It Works

```
Session 1: User talks to agent
  └→ Messages stored to Acontext session (auto-capture)
  └→ CORE extracts structured tasks
  └→ Learning Space distills skills as Markdown

Session 2: User returns
  └→ Skill files + task history injected as context (auto-recall)
  └→ Agent responds with full context
  └→ New messages captured (auto-capture)
  └→ Skills updated (auto-learn)
```

The plugin hooks into two OpenClaw lifecycle events:

- **`before_agent_start`** — Reads learned skill files and past task summaries, injects them into the agent's context via `prependContext`
- **`agent_end`** — Stores conversation messages to Acontext, triggers task extraction, and conditionally triggers skill distillation

## Configuration

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `apiKey` | `string` | — | **Required.** Acontext API key (supports `${ACONTEXT_API_KEY}`) |
| `baseUrl` | `string` | `https://api.acontext.app/api/v1` | Acontext API base URL |
| `userId` | `string` | `"default"` | Scope sessions per user |
| `learningSpaceId` | `string` | auto-created | Explicit Learning Space ID |
| `autoCapture` | `boolean` | `true` | Store messages after each turn |
| `autoRecall` | `boolean` | `true` | Inject skills + tasks before each turn |
| `autoLearn` | `boolean` | `true` | Trigger skill distillation after sessions |
| `maxSkillFiles` | `number` | `10` | Max skill files to inject per recall |
| `maxSkillFileTokens` | `number` | `2000` | Token limit per skill file |
| `maxTaskSummaryTokens` | `number` | `1500` | Token limit for task summaries |
| `recallSessionCount` | `number` | `3` | How many past sessions to recall tasks from |
| `minTurnsForLearn` | `number` | `4` | Minimum conversation turns before triggering auto-learn |

<Tip>
For self-hosted Acontext, set `baseUrl` to your local API endpoint (e.g. `http://localhost:8029/api/v1`).
</Tip>

## Agent Tools

The plugin registers four tools your agent can call explicitly during conversations:

| Tool | Description |
|------|-------------|
| `acontext_search_skills` | Search through learned skill files by keyword |
| `acontext_read_skill` | Read the full content of a specific skill file |
| `acontext_session_history` | Get task summaries from recent past sessions |
| `acontext_learn_now` | Trigger skill learning from the current session immediately |

## CLI Commands

```bash
# List all learned skills in the Learning Space
openclaw acontext skills

# Show memory statistics (sessions, tasks, skills)
openclaw acontext stats
```

## What Gets Recalled

- **Skill files** — Markdown files generated by the Learning Space from past sessions. These contain distilled knowledge: coding preferences, project conventions, learned procedures.
- **Task summaries** — Structured task descriptions and progress from recent sessions, showing what was accomplished.

<Note>
Unlike plugins that extract discrete facts via LLM, Acontext stores full conversations and distills them into human-readable, editable Markdown skill files. You can inspect, modify, and share these files directly.
</Note>
